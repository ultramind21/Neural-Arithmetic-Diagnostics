# PROJECT 10 METHOD REFINEMENT PLAN
## Strengthening Law Testing Beyond Textual Matching

**Date:** April 2026  
**Project:** 10  
**Status:** ACTIVE METHOD REFINEMENT

---

## 1. Why This Plan Exists

The first explicit law test in Project 10 was valuable, but it also exposed a methodological weakness:

- the test relied too much on textual support patterns inside project summaries
- it did not yet operate on sufficiently formalized evidence structures

This means Project 10 must improve its testing method before scaling law evaluation further.

---

## 2. Core Problem

A theory-building project cannot rely mainly on:
- wording overlap
- textual phrasing
- or summary-language cues

Those can provide early signals, but they are not enough for serious law testing.

If Project 10 is to become a genuine theory project, it must move toward:
- structured evidence extraction
- explicit candidate-law mapping
- and falsification-aware evidence tables

---

## 3. Main Goal

The main goal of this refinement phase is:

> turn Project 10 law testing from summary-language matching into structured evidence evaluation.

---

## 4. What Must Improve

### A. Evidence Extraction
For each project, extract:
- direct empirical signals
- accepted result type
- whether the result supports, weakens, or does not address a candidate law

### B. Law Mapping
For each law candidate, define:
- which projects are relevant
- what kind of evidence counts
- what kind of result would weaken it
- what kind of result would falsify it

### C. Boundary Logic
Distinguish:
- support
- partial support
- no evidence
- contradiction
- and true falsification

These are not the same.

---

## 5. Immediate Methodological Upgrade

The next law-testing layer should rely on:
- structured evidence tables
- not free-form text only

This means the next generation of law testing should explicitly record, per project:

1. evidence item
2. law relevance
3. support / weaken / irrelevant
4. confidence level
5. notes on boundaries

---

## 6. Why This Matters

This matters because Project 10 is the first project in the line attempting real abstraction.

If its methods are weak, the whole theory layer becomes weak.

If its methods improve now, the later law results become much more credible.

So this is not a side issue.
It is central.
