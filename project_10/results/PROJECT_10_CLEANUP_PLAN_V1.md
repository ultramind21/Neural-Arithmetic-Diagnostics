**Status:** ACTIVE CURATED CLEANUP PLAN

---

## 1. Purpose

This document defines the curated cleanup plan for Project 10 after reaching a stable packaged checkpoint.

The goal is not destructive cleanup.

The goal is to:

- preserve core scientific assets
- reduce clutter
- separate canonical materials from superseded or unreviewed ones
- and make the project easier to navigate without damaging research integrity

---

## 2. Core Cleanup Principle

Project 10 should be cleaned through:

- **keep**
- **move**
- **archive**
- and only rarely **delete**

Deletion should be used only for files that are clearly disposable or purely local utility artifacts.

Research materials should generally be preserved unless there is a strong reason not to.

---

## 3. Categories

### A. Keep (canonical)
Files that define the official scientific and presentation state of Project 10.

### B. Move to superseded
Earlier versions that remain historically useful but should not dominate the main view.

### C. Move to unreviewed_agent_outputs
Files created outside controlled instruction flow and not treated as canonical.

### D. Move to tools or archive
Utility scripts or local helper files that do not belong in the core Project 10 scientific surface.

### E. Delete
Only for clearly disposable files if they are confirmed unnecessary.

---

## 4. Immediate Cleanup Priorities

### Priority 1
Separate canonical from non-canonical result files.

### Priority 2
Move superseded summaries out of the main results surface.

### Priority 3
Move utility/local files out of the root-level Project 10 view.

### Priority 4
Delete only clearly disposable clutter if confirmed safe.
