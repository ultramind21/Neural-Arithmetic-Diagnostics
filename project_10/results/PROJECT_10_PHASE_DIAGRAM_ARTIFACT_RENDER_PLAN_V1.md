## 1. Purpose

This document defines the rendering decision for the first rescue phase diagram artifact in Project 10.

At this stage, the project already has:

- artifact specification
- official figure text
- layout structure

The remaining question is how far rendering should go in the current phase.

---

## 2. Current Decision Principle

The goal is not to produce a polished publication figure too early.

The goal is to produce a stable, interpretable, theory-facing artifact that:

- captures the current regime-space structure
- remains easy to revise
- and does not over-invest in visual polish before the theory stabilizes further

This is the governing principle.

---

## 3. Recommended Rendering Level

The recommended current rendering level is:

### **lightweight rendered artifact**

This means:
- a structured visual-text artifact
- conceptually clear
- easy to update
- and sufficient for theory communication

It does **not** yet require:
- full publication-grade plotting
- final figure styling
- or irreversible visual commitment

---

## 4. What Should Be Rendered Now

The current artifact should render:

- axes
- region names
- approximate region arrangement
- transition-band annotations
- boundary drift annotation
- short caption

This is enough for the current phase.

---

## 5. What Should Wait

The following should wait until later:

- final publication styling
- polished external-facing figure production
- high-fidelity plotting refinement
- and any full asset-pipeline integration

Those would be premature at the current stage.
