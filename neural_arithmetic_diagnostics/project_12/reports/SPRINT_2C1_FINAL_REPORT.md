## Part 2: Critical Finding — Claim P11-C07 Failure

### Details

**Claim:** Soft clamp (k=15) significantly improves rule performance at boundaries; compact V3.1 achieves near-competitive boundary performance.

**Observed (Phase D boundary subset):**
- V3.1 boundary macroF1_present: **0.759266**
- V3 hard clamp boundary: 0.597485
- NN81 boundary: 0.757786

**Targets (from FORMAL_CLAIMS.md):**
1. V3.1 boundary ≥ 0.85: ❌ **FAIL** (0.759 < 0.85)
2. (V3.1 − V3) boundary ≥ 0.15: ✅ PASS (0.162 ≥ 0.15)
3. (NN81 − V3.1) boundary ≤ 0.10: ✅ PASS (-0.001 ≤ 0.10)

**Analysis:**

The failure is **in the absolute target (0.85), not in the relative improvement**:
- The soft clamp does improve boundary performance relative to hard clamp (gap = 0.162 ≥ 0.15) ✅
- V3.1 is near-competitive with NN81 on boundary (gap = -0.001) ✅
- **BUT:** V3.1 does not reach the canonical absolute threshold of 0.85

**Interpretation:**

This suggests that on the **boundary subset**:
- The holdout generated by Project 12 (seed=424242) produces a more challenging boundary distribution than Project 11's original (seed=223311)
- The soft clamp mechanism works as intended (relative improvement confirmed)
- However, the specific performance plateau point differs from Project 11's original data

**Context from Project 11:**
- Project 11 original V3.1 boundary: 0.8693 ✅ (⊢ ≥ 0.85)
- Project 12 + seed=424242 V3.1 boundary: 0.7593 ❌ (fails threshold)
