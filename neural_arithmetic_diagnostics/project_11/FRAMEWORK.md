# PROJECT 11 FRAMEWORK
## Predictive and Transfer Regime Theory Beyond Project 10

**Date:** April 2026  
**Status:** ACTIVE  
**Entry condition:** Project 11 begins only after Prediction Gate PASS

---

## 1. Purpose

Project 11 escalates the program beyond Project 10.

Project 10 produced a regime/threshold/phase-diagram account of rescue behavior inside a controlled sandbox.
Project 11 asks whether that structure becomes:

- predictive (pre-run predictions, not post-hoc explanations)
- transferable (beyond the original system)
- compressible (compact boundary rules rather than only tables)

---

## 2. Core Question

> Can the regime/threshold structure discovered in Project 10 be used as a predictive engine and transferred beyond its original sandbox?

---

## 3. Critical Constraint — Out-of-Distribution Requirement (MANDATORY)

Project 11 predictions must **NOT** be evaluated on the same generative/evaluation system used in Project 10.

At least one evaluation system must differ in at least one of:
- scoring rules, OR
- failure propagation dynamics, OR
- intervention mechanics

If evaluation uses the same system, results are considered descriptive, not predictive.

---

## 4. Prohibition — Evaluation Leakage (MANDATORY)

Prediction evaluation must not reuse the same functional form used to derive the original Project 10 phase diagram.

If the evaluation function structurally matches the original system, the test is invalid.

---

## 5. What Counts as Success (Project 11)

Project 11 succeeds only if it produces at least two of:

1. **Prediction-before-run success** on holdout regimes  
2. **Transfer** to a second system/task/sandbox with modified rules  
3. **Compression** into a compact predictive rule (curve/inequality/classifier) that generalizes  
4. **Robustness under adversarial changes** (noise, partial observability, mismatch)

---

## 6. Expected Failure Clause (MANDATORY)

Prediction failure is not a negative outcome.

Failure is required to:
- identify limits of the regime theory
- refine boundary definitions
- improve metric design

Perfect accuracy is not expected in early Project 11 phases.

---

## 7. What Project 11 Will Not Do

- No post-hoc prediction disguised as theory
- No "simulate Project 10 again" and call it transfer
- No decorative mathematics without predictive tests
- No expansion into many tasks before one clean transfer test exists

---

## 8. Phase Outline

### Phase A — Operationalization
Lock measurable metrics for:
- local competence saturation (L)
- global robustness (G)
- heterogeneity (H)
- universal rescue power (P)

### Phase B — Holdout prediction (same broad domain, new points)
Pre-register predictions before running.

### Phase C — Transfer test (new system)
Create a modified system that preserves local/global tension but changes failure or intervention dynamics.

### Phase D — Compression
Derive a compact boundary model (e.g., a critical curve \(P_c(H)\)) and test it.
