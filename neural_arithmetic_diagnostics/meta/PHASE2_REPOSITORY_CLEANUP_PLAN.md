# PHASE 2 REPOSITORY CLEANUP PLAN
## Neural-Arithmetic-Diagnostics

**Date:** April 2026  
**Purpose:** Second-pass cleanup after expansion through Projects 5–7

---

## 1. Why This New Cleanup Phase Is Needed

The repository is now much richer than it was after Project 4.

It no longer contains:
- only the original projects
- only the audit
- and only Project 4

It now also contains:
- a paused decomposition branch (Project 5)
- a strong mechanistic platform (Project 6)
- and a local-to-global bridge branch (Project 7)

This makes the repository scientifically stronger, but also structurally heavier.

A second cleanup pass is therefore needed.

---

## 2. Main Goal

The goal of this cleanup phase is:

- preserve the full research wealth
- reduce visual clutter
- make the repository easier to understand
- distinguish accepted cores from paused/exploratory branches
- and improve public-facing readability without losing scientific depth

---

## 3. Problems to Solve

### A. Root-level crowding
Too many top-level files are now visible at once.

### B. Overlapping status files
There are multiple:
- synthesis
- checkpoint
- closure
- current-status
- comprehensive
style files across projects.

### C. Legacy leftovers
Some files remain useful historically but should not stay highly visible.

### D. Inconsistent naming force
Some names reflect live scientific value, others reflect working-process history.

---

## 4. Cleanup Principles

1. Do not delete scientific evidence lightly
2. Move clutter before deleting anything
3. Keep final/high-value summaries visible
4. Archive transitional layers rather than mixing them with final materials
5. Preserve a clean path for first-time visitors

---

## 5. Priority Targets

### Priority 1 — Root cleanup
Review which files should remain top-level visible.

### Priority 2 — Project 5/6/7 result hierarchy
Separate:
- accepted core
- paused status
- exploratory notes
- synthesis layers

### Priority 3 — Legacy technical artifacts
Review helper files such as:
- `generate_diagram.py`
- `unnamed.png`
- temporary or duplicate planning files

### Priority 4 — Naming clarity
Decide where "FINAL", "SYNTHESIS", "CHECKPOINT", "CURRENT_STATUS", and "CLOSURE" are all necessary and where they should be consolidated.

Potential future moves may include:

  - final accepted summary
  - paused branch checkpoint
