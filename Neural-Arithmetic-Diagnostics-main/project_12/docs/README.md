# Project 12 — Validation & Consolidation

**Project 12 is the claim authority layer.**

This project converts Projects 4–11 from a "system of ideas" into a "system of evidence".

Core outputs:
- FORMAL_CLAIMS.md: Locked claims with acceptance criteria.
- EVALUATION_SPEC.md: Unified metrics, baselines, and leakage rules.
- VALIDATION_PROTOCOL.md: The operating contract for reproducible validation.
- RUNBOOK.md: Step-by-step execution checklist (manifest-driven).
- [CLAIM_AUTHORITY.md](CLAIM_AUTHORITY.md): Policy for validating and tracking claim status.

Rules:
- Projects 4–11 are treated as read-only.
- All Project 12 outputs must go under project_12/results/.
- No narrative/hype in validated results — evidence only.
