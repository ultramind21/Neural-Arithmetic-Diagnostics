## Version 1.0
**Date:** March 31, 2026  
**Status:** INITIAL LOCKED FRAMEWORK

### Summary
Initial formal release of the Project 4 diagnostic framework after closure of the primary trust-recovery audit.

### Framework Position
Project 4 begins from a post-audit state:
- primary audit sequence (Phases 1–6) complete
- core trust substantially recovered
- locked caveats preserved
- Project 4 launched as a diagnostic framework project, not as an unrestricted model-improvement race

### Core Regimes Defined
- Regime 1: Distribution-bound fit
- Regime 2: Bounded compositional competence
- Regime 3: Stronger algorithm-like generalization

### Core Diagnostic Families Defined
- In-distribution random
- Structured adversarial
- Length extrapolation
- Mechanism ablations
- Optional extended tests

### Core Scorecard Dimensions Defined
1. In-distribution accuracy
2. Structured adversarial accuracy
3. Length extrapolation trend
4. Rounding sensitivity
5. Carry corruption sensitivity

### Locked v1.0 Rules
- Project 4 does not require finding a Regime 3 model
- No single metric is sufficient for regime classification
- Regime classification is rule-guided, not fully automatic
- Pattern-wise breakdown must be reported
- Worst-case pattern accuracy must be reported
- Negative results remain scientifically valid outcomes
