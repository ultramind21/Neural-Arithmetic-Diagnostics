# REPOSITORY OVERVIEW FIGURE SPEC
## Neural-Arithmetic-Diagnostics

**Target image filename:** `repository_overview.png`

---

## 1. Purpose

This figure is the first public-facing visual asset for the repository.

Its goal is to help a first-time visitor understand, in a few seconds, that the repository contains:

1. a historical research line
2. a verification / audit layer
3. a post-audit diagnostic framework

The figure should be simple, readable, and visually clean.

---

## 2. Figure Title

**Neural-Arithmetic-Diagnostics**  
**Research → Audit → Diagnostic Framework**

---

## 3. Core Layout

Use a left-to-right or top-to-bottom flow with **three main blocks**:

### Block 1 — Projects 1–3
Label:
- **Projects 1–3**
- *Original Research Line*

Short subtext:
- Arithmetic learning experiments
- Killer-test findings
- Historical project closures

---

### Block 2 — Trust-Recovery Audit
Label:

### Block 3 — Project 4
Label:

**Projects 1–3** → **Audit** → **Project 4**

Optional small labels:
- "research"

- "diagnostics"

---

## 5. Bottom Message

At the bottom of the figure, include this short statement:

> **High arithmetic accuracy is not sufficient evidence of robust reasoning.**

---

## 6. Style Requirements

The figure should be:

- clean
- minimal
- GitHub-friendly
- easy to read at moderate zoom
- without clutter
- without tiny text
- visually professional rather than decorative

Preferred style:
- white or light background
- 3 colored blocks
- dark readable text
- simple arrows
- no unnecessary icons unless very subtle

---

## 7. Text to Avoid

Do not overload the figure with:
- many filenames
- many phase details
- long caveat text
- full research claims
- dense bullet lists

This is a visual orientation asset, not a full summary document.
